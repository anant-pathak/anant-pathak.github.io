Types and declarations

What is?

1) "_t" : Is used to denote aliases :

2) "static_assert()" : To check assumptions of implementation defined features. E.g. static_assert(4<=sizeof(int),"sizeof(int) too small"); 3) "static_cast<>" : Use it to safely convert something. E.g. static_cast('0' + i);

4) "ll or LL"suffix for long long || for others.

5) "numeric_limits" : std::numeric_limits::is_signed

Character Type:


6 different types: char ; signed char {0, 255} ; unsigned char {-127, 127} ; wchar_t ; char16_t; char32_t Normally char has 8bits (1Byte) which can only hold ASCII set. NOTE A 'char' could be unsigned or signed.

INT type:

Plain int is always signed int. Unlike 'char' . long long int == long long .

SIZE:

Guaranteed sizes: Char 8 bits Short 16 bits long 32 bits NOT guareenteed: sizeof(long) < sizeof(long long) sizeof(double) < sizeof(long double) size_t : Is an implementation defined "unsigned integer type" that can hold the size of every object in bytes. ptrdiff_t : signed integer type for holding the result of subtracting two pointers to get a number of elements.

Pointers

Declaration types:

1) int* ap[15]; //Array of 15 Pointers to ints.
2) int (*fp) (char*) //Pointer to a Fn taking a char* and returning an int
3) void (Class_name::*fnPtr) (string) = &Class_name::f; //Pointer to a member Fn & how to invoke it. (class_obj_ptr->*fnPtr)('string_arg');

VOID*

Pointer to an object of unknown type.
Note: Occurrences of void*s at higher levels of the system should be viewed with great suspicion because they are likely indicators of design errors.
NOTE: Pointer to a FN & Pointer to a member( ->* ) can't be assigned to a void*
