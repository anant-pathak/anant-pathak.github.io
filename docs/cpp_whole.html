decltype :

 

    • Parrots the type of an object.
    • Used in conjuction with auto and templates.
    • The use of auto before a function name has nothing to do with type deduction. Rather, it indicates that the trailing return type syntax has been used. which is ("->").
    • In a function's return type if you wish to use the parameters itself, than you'd have to use decltype, after their names are declared in the parameter list.
    • Also, note an auto return type with a function deploys Template type deduction not the Auto type deduction.
    • auto in a function return type or a lambda parameter implies template type deduction, not auto type deduction.

unique_ptr<>

 

    • Uses the same space as that of a raw pointer.
    • Copying it isn't allowed, only std::move() is allowed.
    • The resource gets delete when the pointer  is deleted. How? Using 'delete'
    • You can pass in a custom deleter.
    • Std::unique_ptr<T> is for individual objects, it doesn't support the operator '[]'
    • std::unique_ptr<T[]> for ptr to an array, it doesn't support: "*" & "->"
    • For pointing to arrays you should use other container like std::vector , std::array or string.
    • Very suitable for cases of factory method, where the called inherited class is created on heap and returned when requested.
    • A unique_ptr can be converted to a shared_ptr as well by the user:

std::shared_ptr<Investments> sp = makeInvestment(arg); //makeinvestments returns a unique_ptr tho.

    • You can provide your own "deleter" Fn too.
